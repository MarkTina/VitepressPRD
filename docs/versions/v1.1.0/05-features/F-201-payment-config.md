# 支付配置中心

## 功能概述

建立网点级别的支付配置管理平台。支持创建多种支付模式的收款配置（微信服务商分账、微信服务商直收、支付宝直收），并将配置绑定到指定自助洗衣网点，实现网点维度的灵活收款管理。

## 用户故事

> 作为 **平台运营管理员**，我希望 **在后台创建多种支付模式的收款配置并分配给不同网点**，以便 **每个网点都能使用最适合自己的收款方式**。

> 作为 **加盟商**，我希望 **了解自己网点目前使用哪种收款方式和费率**，以便 **清晰掌握经营成本**。

## 前置条件

- 已在微信支付服务商平台完成账号注册和认证
- 已有至少一个微信子商户
- 已在支付宝开放平台完成应用创建

## 功能详述

### 场景 1：创建支付配置（正常流程）

1. 运营管理员进入「支付配置」页面，点击「新建配置」
2. 选择支付模式：
   - 微信服务商分账模式
   - 微信服务商直接收款模式
   - 支付宝直接收款模式
3. 根据所选模式，填写对应的配置参数（见下方各模式详述）
4. 点击「保存」，系统校验参数格式和有效性
5. 配置创建成功，状态为「已启用」

### 场景 2：将支付配置绑定到网点

1. 运营管理员进入「网点管理」→ 选择目标网点 →「支付设置」
2. 从支付配置下拉列表中选择一个已启用的配置
3. 确认绑定，系统校验该配置的支付模式是否适用于当前网点
4. 绑定成功，网点侧支付接口自动使用新配置

### 场景 3：更换网点支付配置

1. 运营管理员进入网点支付设置页
2. 选择新的支付配置，确认切换
3. 系统记录变更日志（旧配置、新配置、操作人、时间）
4. 正在进行的支付不受影响，新发起的支付使用新配置

### 场景 4：停用支付配置

1. 运营管理员进入支付配置详情页，点击「停用」
2. 系统检查该配置是否仍被网点绑定
3. 若有网点绑定，提示「以下网点正在使用该配置，停用将影响其收款：...」
4. 确认后停用，同时自动解绑所有关联网点（并记录日志）

## 交互说明

### 配置列表页

| 元素 | 类型 | 说明 | 规则 |
|------|------|------|------|
| 筛选-支付模式 | 下拉选择 | 微信分账/微信直收/支付宝/全部 | 默认全部 |
| 筛选-状态 | 下拉选择 | 已启用/已停用/全部 | 默认全部 |
| 新建配置 | 主按钮 | 打开新建表单 | 任何管理员可用 |
| 配置列表项 | 卡片/行 | 名称、模式、状态、绑定网点数 | 点击进入详情 |

### 配置详情页

| 元素 | 类型 | 说明 | 规则 |
|------|------|------|------|
| 配置名称 | 文本 | 可编辑 | 必填，全平台唯一 |
| 支付模式 | 只读 | 创建后不可更改 | — |
| 状态开关 | 开关 | 启用/停用 | 停用需确认 |
| 绑定网点列表 | 表格 | 网点名称、位置、绑定时间 | 点击可跳转网点详情 |
| 删除按钮 | 危险按钮 | 删除配置 | 仅未被绑定时可删除 |

## 验收标准

- [ ] F-201-AC01：支持创建/编辑/停用支付配置，三种模式各有独立的参数 Schema
- [ ] F-201-AC02：配置列表支持按模式和状态筛选
- [ ] F-201-AC03：网点可绑定一个支付配置，切换后立即生效
- [ ] F-201-AC04：停用配置自动解绑关联网点并记录日志
- [ ] F-201-AC05：配置变更记录完整可追溯（操作人、时间、变更内容）
- [ ] F-201-AC06：一个网点有且仅有一个启用的支付配置

## 技术备注

- 配置参数采用 JSON 字段存储，不同模式对应不同 Schema，便于扩展新模式
- 配置变更通过 Redis Pub/Sub 或配置中心（如 Nacos）推送，各服务节点实时生效
- 网点-配置绑定关系需加分布式锁，防止并发切换导致不一致
