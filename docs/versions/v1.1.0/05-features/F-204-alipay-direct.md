# 支付宝直接收款模式

## 功能概述

支持支付宝作为独立支付渠道进行收款，作为微信支付的补充。网点可选择支付宝直接收款模式，覆盖偏好使用支付宝的用户群体。

## 用户故事

> 作为 **用户**，我希望 **用支付宝也能在洗衣网点完成支付**，以便 **不用非得用微信支付**。

> 作为 **加盟商**，我希望 **覆盖更多的支付方式**，以便 **不会因为支付方式限制而丢失订单**。

## 前置条件

- 已在支付宝开放平台创建应用并通过审核
- 已获取支付宝商户 PID 和密钥
- 支付配置中心（F-201）已上线

## 功能详述

### 场景 1：创建支付宝收款配置（正常流程）

1. 运营管理员选择「支付宝直接收款模式」创建配置
2. 填写：应用 ID、商户 PID、支付宝公钥、商户私钥、异步通知地址
3. 可选配置：支付描述前缀
4. 保存配置后，系统校验密钥有效性

### 场景 2：用户支付宝支付（正常流程）

1. 用户选择支付宝支付
2. 系统调用支付宝 `alipay.trade.create`（当面付）或 `alipay.trade.app.pay`（APP 支付）
3. 用户完成支付，支付宝异步通知到达
4. 系统验签后更新订单状态，推送支付成功通知

### 场景 3：退款

1. 用户发起退款
2. 系统调用支付宝 `alipay.trade.refund` 接口
3. 支持全额和部分退款
4. 退款完成后更新订单状态

### 场景 4：支付宝与微信双选（边界情况）

1. 网点同时创建了微信和支付宝两套支付配置
2. 当前仅支持一个网点绑定一种支付配置
3. 若需同时支持微信+支付宝，需未来版本支持「组合支付配置」
4. 当前版本建议：默认使用微信配置，待 F-204 上线后，网点可选择切换为支付宝

## 验收标准

- [ ] F-204-AC01：支持支付宝当面付和 APP 支付两种场景
- [ ] F-204-AC02：支付宝异步通知正确处理（验签、幂等）
- [ ] F-204-AC03：支持全额和部分退款
- [ ] F-204-AC04：支付宝配置与微信配置相互独立

## 技术备注

- 支付宝 SDK 与微信支付 SDK 独立引入，降低耦合
- 异步通知需验签，防止伪造回调
- 支付宝当面付二维码有过期时间（通常 2 小时），需在前端做超时处理
