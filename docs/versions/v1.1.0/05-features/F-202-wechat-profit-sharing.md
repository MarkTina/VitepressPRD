# 微信服务商分账模式

## 功能概述

基于微信支付服务商模式，实现支付+自动分账能力。用户支付完成后，资金按预设规则自动分账到平台方、加盟商、设备供应商等接收方账户，减少手工结算工作量。

## 用户故事

> 作为 **平台运营管理员**，我希望 **配置好分账规则后每次支付自动完成分账**，以便 **不再需要月底手工计算和转账给加盟商**。

> 作为 **加盟商**，我希望 **每笔订单完成后能实时看到自己能分到多少钱**，以便 **资金透明，心里有底**。

## 前置条件

- 已在微信支付服务商平台开通分账功能
- 分账接收方已在微信支付侧完成商户注册
- 支付配置中心（F-201）已上线

## 功能详述

### 场景 1：配置分账规则（正常流程）

1. 运营管理员在创建「微信服务商分账模式」配置时，填写分账规则
2. 分账规则支持两种类型：
   - **按比例分账**：如平台抽 15%，加盟商得 85%
   - **按固定金额分账**：如每单平台抽 2 元，其余归加盟商
3. 添加分账接收方（最多 10 个），每个接收方绑定一个分成比例或固定金额
4. 系统校验所有接收方分成比例之和 ≤ 100%（或固定金额之和 ≤ 订单金额）
5. 保存配置，分账规则生效

### 场景 2：支付+分账（正常流程）

1. 用户在洗衣网点扫码支付，订单金额 20 元
2. 支付请求携带分账标记，微信支付成功后将订单标记为「可分账」
3. 系统调用微信分账接口，按配置的分账规则发起分账：
   - 平台方：20 × 15% = 3 元
   - 加盟商：20 × 85% = 17 元
4. 分账成功后，更新订单分账状态为「已分账」
5. 推送分账结果通知给加盟商

### 场景 3：分账失败处理

1. 微信分账接口返回失败（如接收方账户异常、余额不足）
2. 该笔订单分账状态标记为「分账失败」
3. 系统自动加入分账重试队列，每 30 分钟重试一次，最多重试 5 次
4. 5 次重试均失败后，系统发出告警（站内通知 + 企业微信/钉钉）
5. 财务人员手动处理：核实接收方账户 → 修正 → 手动重试分账

### 场景 4：分账回退

1. 订单发生退款，需要回退已分账的资金
2. 调用微信分账回退接口，按原分账比例退回各接收方
3. 回退成功后更新订单状态，记录回退流水

## 交互说明

### 分账规则配置

| 元素 | 类型 | 说明 | 规则 |
|------|------|------|------|
| 分账方式 | 单选 | 按比例 / 按固定金额 | 必选 |
| 接收方列表 | 动态表单 | 最多 10 个接收方 | 每个接收方：类型、商户号、比例或金额 |
| 接收方类型 | 下拉 | 平台/加盟商/供应商/其他 | 平台必须至少一个 |

### 分账状态

- **待分账**：支付成功，等待发起分账
- **分账中**：已发起分账请求，等待微信返回结果
- **已分账**：全部分账成功
- **部分分账**：部分接收方分账成功（存在失败的接收方）
- **分账失败**：全部分账失败
- **已回退**：分账已回退（退款场景）

## 验收标准

- [ ] F-202-AC01：支持按比例和固定金额两种分账规则
- [ ] F-202-AC02：分账接收方最多 10 个，比例之和校验正确
- [ ] F-202-AC03：支付成功后自动发起分账，正常情况 30 秒内完成
- [ ] F-202-AC04：分账失败自动重试 5 次，5 次后告警
- [ ] F-202-AC05：支持分账查询和分账回退
- [ ] F-202-AC06：分账流水完整可追溯

## 技术备注

- 分账接口调用需加分布式锁，防止同一笔订单并发分账
- 微信分账 API 单次最多 10 个接收方，需做前端限制
- 分账结果回调需做幂等处理
