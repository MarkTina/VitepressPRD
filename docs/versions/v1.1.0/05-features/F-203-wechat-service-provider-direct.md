# 微信服务商直接收款模式

## 功能概述

基于微信支付服务商模式，实现不涉及分账的直接收款。服务商旗下的子商户直接向用户收款，适用于平台不参与分账、仅收取固定服务费或加盟商独立经营的场景。

## 用户故事

> 作为 **加盟商**，我希望 **用户付的款直接进我的商户账户**，以便 **资金即时到账，不用等平台结算**。

> 作为 **平台运营管理员**，我希望 **支持部分网点选择直收模式（不参与分账）**，以便 **灵活适配不同类型加盟商的合作方式**。

## 前置条件

- 已在微信支付服务商平台开通服务商支付权限
- 子商户已完成进件或已有可用子商户号
- 支付配置中心（F-201）已上线

## 功能详述

### 场景 1：创建直收模式配置（正常流程）

1. 运营管理员选择「微信服务商直接收款模式」创建配置
2. 填写服务商商户号、API 密钥、子商户号
3. 可选配置：支付后回调 URL、支付描述前缀
4. 保存配置

### 场景 2：用户支付（正常流程）

1. 用户在洗衣网点扫码支付
2. 系统根据网点绑定的支付配置，获取服务商参数和子商户号
3. 调用微信服务商支付 API（JSAPI / NATIVE），`sub_mch_id` 设置为对应子商户
4. 用户完成支付，资金直接进入子商户账户
5. 系统记录支付流水，推送支付成功通知

### 场景 3：退款

1. 用户发起退款申请
2. 系统调用微信服务商退款接口，退款从子商户账户原路退回
3. 支持全额退款和部分退款
4. 退款完成后更新订单状态

## 验收标准

- [ ] F-203-AC01：支持服务商模式下子商户直接收款
- [ ] F-203-AC02：支付成功回调准确通知到业务系统
- [ ] F-203-AC03：支持全额和部分退款
- [ ] F-203-AC04：与分账模式可在不同网点独立使用，互不影响

## 技术备注

- 服务商模式 API 与直连模式不同，需独立封装 WechatServiceProviderPayAdapter
- 子商户号需在微信服务商后台完成进件后方可使用
- 退款资金从子商户账户出，非服务商账户
