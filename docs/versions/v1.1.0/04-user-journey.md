# 4. 用户旅程

## 旅程地图 A：运营管理员 - 配置网点支付模式

### 阶段 1：进入配置中心

| 触点 | 用户行为 | 用户想法 | 情绪 |
|------|----------|----------|------|
| 后台导航 | 点击「支付配置」菜单 | 今天要为新网点配个分账模式 | 😐 中性 |
| 配置列表 | 浏览已有的支付配置 | 已经有几个配置了，看看有没有能复用的 | 🤔 思考 |

### 阶段 2：创建支付配置

| 触点 | 用户行为 | 用户想法 | 情绪 |
|------|----------|----------|------|
| 新建表单 | 选择「微信服务商分账」模式 | 这个模式适合需要平台抽成的场景 | 😊 积极 |
| 参数填写 | 填入商户号、API 密钥、分账规则 | 参数有点多，但有提示，还行 | 😐 中性 |
| 保存 | 点击保存，校验通过 | 配置创建成功了 | 😊 积极 |

### 阶段 3：绑定网点

| 触点 | 用户行为 | 用户想法 | 情绪 |
|------|----------|----------|------|
| 网点列表 | 找到目标网点，点击「支付设置」 | 就是这个新开的店 | 😐 中性 |
| 绑定操作 | 选择刚创建的配置，确认绑定 | 选这个分账配置就行 | 😊 积极 |
| 绑定成功 | 系统提示绑定成功 | 搞定，网点可以正常收款了 | 😊 积极 |

### 阶段 4：验证

| 触点 | 用户行为 | 用户想法 | 情绪 |
|------|----------|----------|------|
| 测试支付 | 在网点设备发起 0.01 元测试支付 | 确认下支付链路是否通 | 🤔 期待 |
| 查看记录 | 支付成功，查看分账结果 | 平台和加盟商的分账金额都对 | 😊 积极 |

## 旅程地图 B：研发工程师 - 接入短信微服务

### 阶段 1：了解服务

| 触点 | 用户行为 | 用户想法 | 情绪 |
|------|----------|----------|------|
| API 文档 | 阅读短信微服务接口文档 | API 设计很清晰，和公司其他服务风格一致 | 😊 积极 |
| 示例代码 | 查看 SDK/调用示例 | 有现成的 SDK，直接用就行 | 😊 积极 |

### 阶段 2：申请接入

| 触点 | 用户行为 | 用户想法 | 情绪 |
|------|----------|----------|------|
| 租户申请 | 在短信服务管理后台申请 API Key | 跟运营说一声就能拿到 Key | 😐 中性 |
| 配置配额 | 运营分配日配额 5000 条 | 日发 5000 条够用了 | 😊 积极 |

### 阶段 3：开发集成

| 触点 | 用户行为 | 用户想法 | 情绪 |
|------|----------|----------|------|
| 写代码 | 调用 `send` 接口，传入模板 ID 和变量 | 一个 POST 请求就搞定，简单 | 😊 积极 |
| 测试 | 发送测试短信，手机收到 | 没问题，可以上线 | 😊 积极 |

### 阶段 4：上线观察

| 触点 | 用户行为 | 用户想法 | 情绪 |
|------|----------|----------|------|
| 监控面板 | 观察发送成功率和延迟 | 成功率 99.5%，延迟 < 2s，达标 | 😊 积极 |
| 告警 | 某天收到配额即将用尽的提醒 | 及时申请扩容，没影响业务 | 😐 中性 |

## 痛点与机会

| 阶段 | 痛点 | 机会 |
|------|------|------|
| 支付配置 | 每次新增支付模式需研发写代码 | 配置中心支持插件化扩展支付模式 |
| 网点绑定 | 网点数与支付配置管理不成比例 | 支持批量绑定、配置继承（默认配置） |
| 短信接入 | 新业务接入短信需走申请流程 | 自助申请 + 自动审批 + 默认配额 |
| 对账 | 多渠道对账手工汇总 | 统一对账面板，自动拉取微信/支付宝账单 |
| 分账异常 | 分账失败后需手动排查 | 自动重试 + 失败原因分类 + 处理建议 |
