## 风险说明

### Risk-001：VitePress 版本升级导致插件不兼容

**风险描述：** VitePress 或 Mermaid 插件升级可能引入 breaking changes，导致构建失败。

**风险等级：** 中

**影响范围：** 构建系统、文档站点部署

**应对方案：**
1. 在 package.json 中锁定主要版本号
2. 升级前在测试分支验证
3. 关注官方 release note
4. 保留可回滚的 commit

**责任人：** 维护者

### Risk-002：版本目录过多导致性能下降

**风险描述：** 随着版本数量增加，构建时间和站点体积可能增大。

**风险等级：** 低

**影响范围：** 构建速度、部署体积

**应对方案：**
1. 归档 6 个月前的版本到单独分支
2. 定期评估是否需要保留历史版本
3. 使用增量构建

**责任人：** 维护者

### Risk-003：AI 生成的 PRD 质量不稳定

**风险描述：** AI 生成的 PRD 可能存在遗漏、不一致或不可行的问题。

**风险等级：** 中

**影响范围：** 文档质量、研发效率

**应对方案：**
1. 建立强制审校环节（阶段 3）
2. 产品经理对最终 PRD 负责
3. 持续优化 Prompt 模板
4. 建立 PRD 质量 checklist

**责任人：** 产品经理

### Risk-004：团队成员不熟悉 Markdown

**风险描述：** 新成员可能不熟悉 Markdown 语法，影响 PRD 撰写效率。

**风险等级：** 低

**影响范围：** 使用门槛、上手速度

**应对方案：**
1. 提供 Markdown 基础教程链接
2. 模板文件作为最佳实践参考
3. AI 辅助生成减少手写 Markdown

**责任人：** 维护者

### Risk-005：原型截图管理混乱

**风险描述：** 原型截图文件多、命名不规范，导致难以查找和关联。

**风险等级：** 低

**影响范围：** 文档维护、协作效率

**应对方案：**
1. 强制使用素材索引表登记
2. 统一命名规范：`{编号}-{功能}-{描述}.png`
3. 定期清理废弃素材

**责任人：** 产品经理
