# 4. 用户旅程

## 旅程地图：产品经理撰写 PRD

### 阶段 1：准备

| 触点 | 用户行为 | 用户想法 | 情绪 |
|------|----------|----------|------|
| 接到需求 | 阅读需求背景 | 这个需求涉及哪些方面？ | 😐 中性 |
| 打开脚手架 | 运行 `npm run new-version` | 新版本的目录已经准备好了 | 😊 积极 |

### 阶段 2：需求讨论

| 触点 | 用户行为 | 用户想法 | 情绪 |
|------|----------|----------|------|
| 与 AI 对话 | 描述需求想法 | AI 的问题帮我理清了思路 | 😊 积极 |
| 回答追问 | 补充细节信息 | 原来这个细节很重要 | 🤔 思考 |
| 确认需求 | 查看结构化输出 | 需求要点很清晰 | 😊 积极 |

### 阶段 3：PRD 生成

| 触点 | 用户行为 | 用户想法 | 情绪 |
|------|----------|----------|------|
| 发起生成 | 发送需求要点给 AI | 期待 AI 的输出 | 🤔 期待 |
| 逐章审核 | 阅读 AI 生成的章节 | 这章写得不错，那章需要改 | 😊/😐 混合 |
| 提出修改 | 告诉 AI 修改意见 | AI 理解了我的意思 | 😊 积极 |

### 阶段 4：审校与确认

| 触点 | 用户行为 | 用户想法 | 情绪 |
|------|----------|----------|------|
| 发起审校 | 让 AI 检查完整 PRD | 希望没有遗漏 | 😐 中性 |
| 查看报告 | 阅读审校问题列表 | 这些问题确实需要改 | 😊 积极 |
| 修改完善 | 根据建议修改 | PRD 质量有保证了 | 😊 积极 |

### 阶段 5：交付

| 触点 | 用户行为 | 用户想法 | 情绪 |
|------|----------|----------|------|
| 提交 Git | `git commit` + `git merge` | 版本历史留痕了 | 😊 积极 |
| 分享链接 | 发送文档站点给研发 | 大家都能看到最新 PRD | 😊 积极 |
| 用户文档 | 让 AI 生成用户说明 | 一份 PRD 多种用途 | 😊 积极 |

## 痛点与机会

| 阶段 | 痛点 | 机会 |
|------|------|------|
| 准备 | 每次创建 PRD 都要从零开始 | 脚手架提供标准化模板 |
| 需求讨论 | 需求想不清楚，遗漏关键点 | AI 追问帮助完善思考 |
| PRD 生成 | 撰写文档耗时，格式不统一 | AI 按模板自动生成 |
| 审校 | 自查容易遗漏 | AI 多维度检查 |
| 交付 | 文档分散各处 | 统一站点集中管理 |
