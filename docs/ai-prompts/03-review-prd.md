# 阶段 3：PRD 审校

## Prompt 模板

```
请对以下 PRD 进行审校检查。

【PRD 内容】
{将已生成的 PRD 各章节内容粘贴至此}

请从以下维度进行检查：

### 1. 完整性检查
- [ ] 是否包含全部 11 项标准要素
- [ ] 每个要素是否有实质性内容（非空模板）
- [ ] 功能模块是否有对应的验收标准
- [ ] 流程图是否覆盖了所有主流程和异常流程

### 2. 一致性检查
- [ ] 术语在全文中是否一致
- [ ] 需求 ID 是否有重复或遗漏
- [ ] 版本号、日期等元信息是否正确
- [ ] 链接是否有效

### 3. 可行性检查
- [ ] 技术方案是否可行
- [ ] 资源投入是否合理
- [ ] 时间估算是否现实
- [ ] 依赖关系是否清晰

### 4. 质量检查
- [ ] 需求描述是否清晰无歧义
- [ ] 验收标准是否可测试
- [ ] 边界情况是否考虑充分
- [ ] 异常处理是否完整

请输出：
1. 检查总结（通过/需修改/严重问题）
2. 具体问题列表（按严重程度排序）
3. 修改建议
```

## 审校清单

### 完整性

| 要素 | 检查点 |
|------|--------|
| 版本首页 | 元信息完整、变更日志清晰 |
| 项目背景 | 有数据支撑、目标明确 |
| 核心需求 | 每个需求有 ID、优先级、描述、验收标准 |
| 用户画像 | 有具体的人物特征、场景描述 |
| 用户旅程 | 覆盖关键触点、有情绪曲线 |
| 功能模块 | 每个功能有概述、详述、验收标准 |
| 流程图 | 主流程 + 异常流程完整 |
| 原型素材 | 有索引说明、关联到具体功能 |
| 数据模型 | 实体关系清晰、字段定义完整 |
| 接口定义 | 请求/响应格式明确 |
| 验收标准 | 可测试、可验证 |
| 风险评估 | 有风险等级、应对方案 |

### 一致性

- [ ] 术语表是否建立并遵循
- [ ] 命名规范是否统一
- [ ] 前后引用是否一致

### 可行性

- [ ] 技术依赖是否可获取
- [ ] 团队能力是否匹配
- [ ] 时间资源是否充足
