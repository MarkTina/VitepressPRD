# 阶段 1：需求讨论

## Prompt 模板

```
我现在有一个产品需求想法，想请你帮我梳理清楚。

【需求描述】
{用户输入的需求描述}

请你帮我：

1. 明确这个需求的背景（为什么做）
2. 确定目标用户是谁
3. 梳理核心使用场景（至少 3 个）
4. 列出所有功能点（按优先级 P0/P1/P2 分类）
5. 识别潜在的边界情况和异常场景
6. 评估技术可行性（大致判断，不需要详细方案）

请用结构化的方式输出，每个部分都要具体、可执行，不要泛泛而谈。

如果需求描述中有不清楚的地方，请先提出追问，帮助我澄清。
```

## 使用说明

1. 将 `{用户输入的需求描述}` 替换为实际的需求描述
2. 发送给 AI 后，根据 AI 的追问补充信息
3. 多轮对话直到需求完全清晰
4. 将最终的结构化需求保存，作为阶段 2 的输入

## 输出格式

```markdown
## 需求梳理结果

### 1. 需求背景
{背景说明}

### 2. 目标用户
{用户画像}

### 3. 核心场景
- 场景 1：{场景描述}
- 场景 2：{场景描述}
- 场景 3：{场景描述}

### 4. 功能清单
#### P0（必须有）
- {功能 1}
- {功能 2}

#### P1（应该有）
- {功能 3}

#### P2（可以有）
- {功能 4}

### 5. 边界与异常
- {边界情况 1}
- {异常场景 1}

### 6. 技术可行性评估
{评估结论}
```
