# 阶段 4：用户文档生成

## Prompt 模板

```
请基于以下已确认的 PRD，生成面向终端用户的操作说明文档。

【PRD 内容】
{将已确认的 PRD 内容粘贴至此}

要求：

1. 目标读者：产品的最终用户（非技术人员）
2. 语言风格：简洁易懂，避免技术术语
3. 内容结构：
   - 功能简介（这是什么功能，能解决什么问题）
   - 使用前提（需要什么权限/条件）
   - 操作步骤（分步骤说明，每步配文字描述）
   - 常见问题（用户可能遇到的问题和解决方法）

4. 不要包含：
   - 技术实现细节
   - 接口定义
   - 数据模型
   - 内部流程

5. 输出格式：Markdown，可直接放入 PRD 版本的 12-user-doc.md

请确保文档对用户友好，让没有技术背景的用户也能轻松理解。
```

## 输出位置

生成的用户文档应放置在对应版本的目录下：

```
versions/
└── vX.Y.Z/
    └── 12-user-doc.md    # 用户操作说明文档
```

## 文档结构模板

```markdown
---
title: 功能使用指南
version: vX.Y.Z
---

# 功能使用指南

## 简介

{功能简介}

## 使用前提

- {前提条件 1}
- {前提条件 2}

## 操作步骤

### {功能名称}

1. **步骤 1**：{操作说明}
2. **步骤 2**：{操作说明}
3. **步骤 3**：{操作说明}

## 常见问题

### Q1：{问题}

A：{回答}

### Q2：{问题}

A：{回答}
```
